Part V - Browser Requests (Network Tab)

1. File retrieval order (HTML, CSS, image)

When I open mysite.html, the browser first sends an HTTP request for the HTML file. After receiving the HTML, the browser parses it and discovers linked resources. It then requests css/responsive.css because of the <link rel="stylesheet"> tag, and requests the image file because of the <img src="..."> tag. CSS and images may be downloaded in parallel, but they are triggered only after the HTML is received and parsed.

2. Why fetch HTML before CSS and images?

The HTML file is the entry point. Before the browser downloads the HTML, it does not know which CSS file or images the page needs. The references to CSS and images are inside the HTML, so the browser must fetch and parse the HTML first to learn which additional resources to request.

3. What does Status Code 200 OK mean for responsive.css?

Status Code 200 OK means the request was successful. The server found responsive.css and returned its contents correctly, so the browser can apply the stylesheet to render the page with the intended layout and design.
